## Project Overview

This is a Kubernetes Operator built with Kubebuilder v4 that translates unbind Service CRDs into native Kubernetes resources (Deployments, Services, Ingress, Databases). It auto-installs dependent operators (MySQL via MOCO, ClickHouse, PostgreSQL via Zalando) using Flux Helm Controller.

## Common Commands

```bash
# Build and test
make build              # Build manager binary to bin/manager
make test               # Run unit tests with coverage
make test-e2e           # Run e2e tests (requires Kind cluster)
make lint               # Run golangci-lint
make lint-fix           # Run linter with auto-fixes

# Code generation (run after modifying api/v1/*_types.go)
make manifests          # Generate CRDs, RBAC, webhook configs
make generate           # Generate DeepCopy methods

# Local development
make run                # Run controller from host (uses kubeconfig)
make install            # Install CRDs into cluster
make deploy             # Deploy controller to cluster

# Docker
make docker-build       # Build Docker image
make docker-buildx      # Build multi-platform image (amd64, arm64)
```

## Architecture

```
cmd/main.go                    # Entry point - initializes controller-runtime manager
api/v1/service_types.go        # Service CRD type definitions
internal/controller/
  service_controller.go        # Main reconciliation logic
internal/resourcebuilder/
  resourcebuilder.go           # Factory - NewResourceBuilder(service, scheme)
  deployment.go                # Builds Kubernetes Deployments
  service.go                   # Builds Services
  ingress.go                   # Builds Ingress objects
  database.go                  # Builds Database objects (MySQL, ClickHouse, Postgres)
internal/operator/manager.go   # Auto-installs dependent operators via Flux Helm
config/                        # Kustomize manifests for deployment
```

## Key Patterns

**Resource Building**: Use `ResourceBuilderInterface` and its implementations in `internal/resourcebuilder/` to translate Service CRDs to Kubernetes objects.

**Reconciliation**: The `ServiceReconciler.Reconcile()` method in `service_controller.go` handles all Service CRD changes. Uses finalizer `service.unbind.unbind.app/finalizer` for cleanup.

**Testing**: Uses Ginkgo/Gomega with envtest (spins up local etcd/kube-apiserver). Mocks generated by mockery.

**Labeling**: Immutable base labels (app.kubernetes.io/*) plus custom labels (unbind-team, unbind-project, unbind-service, unbind-environment, unbind-deployment).

## Development Workflow

1. Modify CRD types in `api/v1/service_types.go`
2. Run `make manifests generate` to regenerate code
3. Update controller logic in `internal/controller/service_controller.go`
4. Run `make lint-fix` to auto-format
5. Run `make test` before committing
